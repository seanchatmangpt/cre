## Troubleshooting Guide

### Common Issues and Solutions

**Issue**: Wrong pattern selection
- **Solution**: Use decision tree from reference card
- **Resource**: Pattern selection flowchart
- **Practice**: Pattern identification exercises

**Issue**: Token flow problems
- **Solution**: Verify preset/postset relationships
- **Resource**: Petri net visualization tools
- **Practice**: Trace token step by step

**Issue**: Engine configuration errors
- **Solution**: Check gen_pnet behavior implementation
- **Resource**: [API Reference](../COMPLETE_API_REFERENCE.md)
- **Practice**: Start with test fixture modules

**Issue**: Data-driven routing fails
- **Solution**: Verify data conditions in modes/3
- **Resource**: [Data Patterns Guide](../patterns/Data_PATTERNS.md)
- **Practice**: Simple data routing examples

**Issue**: Resource allocation deadlocks
- **Solution**: Implement proper synchronization points
- **Resource**: [Resource Patterns](../patterns/RESOURCE_PATTERNS.md)
- **Practice**: Two-resource coordination scenarios

### Study Tips
1. **Pattern-first approach**: Learn patterns, not just API calls
2. **Visual thinking**: Draw Petri nets for complex workflows
3. **Incremental building**: Start simple, add complexity gradually
4. **Real projects**: Apply patterns to actual use cases
5. **Error analysis**: Learn from engine failures
6. **Community engagement**: Share patterns and get feedback
7. **Documentation**: Document each pattern implementation
8. **Performance tracking**: Measure and optimize execution time
